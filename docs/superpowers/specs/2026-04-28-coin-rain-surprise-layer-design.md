**日期**：2026-04-28
**作者**：Claude（与用户头脑风暴产出）
**状态**：待用户评审 → 实施计划

## 1. 动机

V2 的金币雨已经能稳定弹出"今日到账 ¥X"+ 老虎机滚动 + 真实金币声 + 6 款 PNG 混合下落。但每天的画面**几乎一模一样**——同样的文案、同样的金币组合。"仪式感"有了，"惊喜"还少。

V3 在不破坏核心 ritual（"今日到账"那一刻的稳定锚点）的前提下，让每次触发的副标 / 视觉 / 金币品种都有概率性变化，加上节日和里程碑的特殊招呼，目标是让用户**每天打开都期待今天会冒什么花样**。

## 3. 中央副标的优先级引擎

中央 `¥X` 大字下方那行小字，按以下优先级取**第一个命中**的（不堆叠，最多显示一行）：

| 优先级 | 类型 | 命中条件 | 文案模板 |
|---|---|---|---|
| 1 | 累计金额节点 | 见 §4 | `累 计 突 破  ¥{amount}` |
| 2 | 节日（含前一天）| 见 §5 | `预 祝 X 快 乐` / `X 快 乐 ✦` |
| 3 | 天数节点 | 见 §6 | `陪 你 赚 了  N 周/月/年` |
| 4 | 隐藏日期彩蛋 | 见 §7 | 各日期专属文案 |
| 5 | 默认随机文案 | 总命中（兜底）| 从池里抽一句 |

**冲突处理示例**：
- 春节当天 + 第 30 天 → 累计 #1 命中："累计突破 ¥XXX"；如果累计未命中，节日 #2："春节快乐 ✦"，天数 #3 让位
- 11/11 + 第 100 天 → 优先级看具体哪个命中。100 天是天数节点 #3，11/11 是隐藏彩蛋 #4，所以"陪你赚了 X 个月"赢

接口：

```python
# surprise.py
def compute_subtitle(*, days_since_install: int, today: date,
                     daily_income: int, intensity: str) -> str
```

## 4. 累计金额节点（优先级 #1）

### 4.1 公式

设：
- `I` = 用户配置的日收入
- `N = floor(log10(10 * I))` —— 第一次节点的指数级别

**节点值**：
- 第 1 次：`10^N`（数学上保证在前 10 天内命中）
- 第 2 次：`10^(N+1)`（在 10–100 天之间命中）
- 之后每次：在 #2 基础上**每涨一个 `10^(N+1)` 的整数倍**触发（`2·10^(N+1)`、`3·10^(N+1)`、...）

### 4.2 例子

| 日入 I | N | 第 1 次 | 第 2 次 | 之后每涨 |
|---|---|---|---|---|
| 50  | 2 | ¥100（第 2 天）| ¥1,000（第 20 天）| ¥1,000 |
| 200 | 3 | ¥1,000（第 5 天）| ¥10,000（第 50 天）| ¥10,000 |
| 500 | 3 | ¥1,000（第 2 天）| ¥10,000（第 20 天）| ¥10,000 |
| 3000 | 4 | ¥10,000（第 4 天）| ¥100,000（第 34 天）| ¥100,000 |

### 4.3 命中检测

每次金币雨触发时，计算：
- `total_today = days * I`（含今天）
- `total_yesterday = (days - 1) * I`

如果 `total_today >= 节点值` 且 `total_yesterday < 节点值`，则今天命中该节点。

**多次模式的命中触发位置**：放在**当天最后一次**触发（`nth == total`），避免一天里反复刷里程碑。

### 4.4 文案

```
累 计 突 破   ¥{amount with thousand-separator}
```

例：`累 计 突 破   ¥10,000` / `累 计 突 破   ¥100,000`

### 4.5 用户改 income 怎么办

每次按**当前 income** 重新计算节点。如果用户中途从 ¥200 改成 ¥3000，新的 N 变 4，可能会出现"累计已经早早超过 ¥10000，但今天'第一次'按 N=4 算节点为 ¥10,000"。这种 edge case 可以接受——按"playfulness"基调，里程碑触发位置近似即可，不追求精确。

## 5. 节日表（优先级 #2）

### 5.1 法定假日（前一天 + 当天双触发）

| 节日 | 公历 | 前一天文案 | 当天文案 |
|---|---|---|---|
| 元旦 | 1/1 | `预 祝 元 旦 快 乐` | `元 旦 快 乐` |
| 春节 | 农历正月初一 | `预 祝 春 节 快 乐` | `春 节 快 乐  ✦` |
| 清明 | 4/4 或 4/5 | `预 祝 清 明 假 期` | `清 明 节 安 康` |
| 劳动节 | 5/1 | `预 祝 五 一 快 乐` | `劳 动 节 快 乐` |
| 端午 | 农历五月初五 | `预 祝 端 午 安 康` | `端 午 安 康` |
| 中秋 | 农历八月十五 | `预 祝 中 秋 团 圆` | `中 秋 快 乐` |
| 国庆 | 10/1 | `预 祝 国 庆 快 乐` | `国 庆 快 乐  ✦` |

### 5.2 不放假但全民有感（只当天，无前一天）

| 节日 | 公历 | 文案 |
|---|---|---|
| 圣诞 | 12/25 | `圣 诞 快 乐  ★` |
| 万圣 | 10/31 | `万 圣 节 快 乐  🎃` |

### 5.3 农历日期硬编码

`surprise.py` 里有一个 dict：

```python
LUNAR_HOLIDAYS = {
    "spring_festival": {  # 农历正月初一
        2026: date(2026, 2, 17),
        2027: date(2027, 2, 6),
        2028: date(2028, 1, 26),
        ...
        2035: date(2035, 2, 8),
    },
    "dragon_boat": { ... },     # 农历五月初五
    "mid_autumn": { ... },      # 农历八月十五
}

QINGMING = {  # 清明 4/4 或 4/5 逐年
    2026: date(2026, 4, 5),
    ...
}
```

数据范围：**2026–2035 共 10 年**。10 年后用户应该早就不用了，到时候补即可。

### 5.4 命中检测

- 当天检测：`today == 节日日期`
- 前一天检测：`today == 节日日期 - timedelta(days=1)`

注意：**前一天提醒只对法定假日做**，圣诞 / 万圣不放假，没必要。

## 6. 天数节点（优先级 #3）

`days_since_install` 命中以下任一即触发：

| 命中 | 文案 |
|---|---|
| `days == 7` | `陪 你 赚 了  1 周` |
| `days % 30 == 0` (i.e., 30, 60, 90, ...) | `陪 你 赚 了  {days // 30} 个 月` |
| `days % 365 == 0` (i.e., 365, 730, ...) | `陪 你 赚 了  {days // 365} 年` |

三档**独立触发**：第 360 天放"12 个月"，5 天后第 365 天再放"1 年"，不抵消。如果某天同时命中"月"和"年"（理论上不会，因为 360 ≠ 365），按"年"为准。

## 8. 幸运币（独立视觉彩蛋，不走副标优先级）

幸运币是**独立机制**，与 §3 的副标优先级引擎完全无关。每场金币雨在 `start()` 时独立掷骰，**3% 概率** 触发：

- **大小**：`COIN_DIAMETER_MAX × 1.8`（比正常最大的还大 80%）
- **位置**：从屏幕水平正中、顶端略上生成（`x ≈ screen_w / 2`，`y ≈ -120`）
- **速度**：略慢于普通金币（`vy_init × 0.7`），让它在画面里多停留 ~1 秒
- **金光晕**：绘制时在 PNG 周围多画一层径向高斯发光（金色 alpha 60 → 0）
- **随机币种**：从 6 款 PNG 里随机选一款贴图

幸运币命中时，**额外多出一行专属小字**显示在中央 ¥X 大字**上方**（注意是上方）：

```
✦   接  住  一  枚  幸  运  币   ✦
```

这行字和下方的副标（"今日到账"+ §3 引擎选出的那行）位置不同、互不冲突。可以同时存在：上方幸运币提示 + ¥X + "今日到账" + 下方副标。

## 9. 金币品种模式（每次触发独立掷骰）

每次 `CoinRainWindow.start()` 时随机决定本场雨的品种来源：

- **80% 单一币种**：从 6 款里随机抽一个 PNG，本场所有金币都用这一款
- **20% 混合**：6 款均匀随机，每枚金币各自独立抽（≈ 当前 V2 行为）

**多次模式**也是 per-rain-trigger 独立掷骰，不是按天锁定。即同一天里 3 场雨，可能第 1 场全大唐、第 2 场混合、第 3 场全龙洋。

实现：`CoinRainWindow.set_intensity()` 之后再加一个 `_pick_rain_mode()`，决定 `self._coin_mode` ∈ `{single_idx, mixed}`。`_make_coin()` 内根据 `_coin_mode` 选 `style_idx`。

Hero coin（配置窗口左下装饰）保持 datang.png，不受此规则影响。

## 11. 接口与数据流

```
coin_rain.py (--rain --nth N --total T)
   │
   ├─ Config.load()
   │
   ├─ today = date.today()
   ├─ days_since_install = (today - cfg.installed_at.date()).days + 1
   ├─ is_last_trigger = (N == T)
   │
   ├─ subtitle = surprise.compute_subtitle(
   │       days_since_install=days_since_install,
   │       today=today,
   │       daily_income=cfg.income,
   │       is_last_trigger=is_last_trigger,    # 累计节点只在最后一次触发
   │   )
   │
   ├─ visual = surprise.compute_visual_overrides(today=today)
   │       # → {"flip_chance": 0.3} 或 {"size_scale": 0.5} 或 {}
   │
   ├─ lucky = random.random() < 0.03
   │
   ├─ w = CoinRainWindow()
   ├─ w.set_subtitle(subtitle)
   ├─ w.set_visual_overrides(visual)
   ├─ w.set_lucky_coin(lucky)
   ├─ w.set_coin_mode(_pick_mode())   # 80/20
   └─ w.start()
```

`surprise.py` 是纯 Python（无 Qt），所有规则集中。

## 13. 不做（YAGNI）

- 不持久化 history.json，所有累计/天数从 `installed_at` 现算
- 不自定义生日 / 发薪日（用户明确不要）
